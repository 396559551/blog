    Vue响应式原理

    张子涵 8/22/2021 Vue

    # vue响应式原理

    1. vue怎么知道数据改变
    2. vue在数据改变时，怎么知道通知哪些视图更新
    3. vue在数据改变时，视图怎么知道什么时候更新

    下面就会涉及到三个重要的概念：

    • Object.defineProperty
    • 依赖收集
    • 依赖更新

    # Object.defineProperty

    Objiect.defineProperty可以为对象的每一个属性，设置get和set方法

    那么 get 和 set 方法有什么用？

    get 值是一个函数，当属性被访问时，会触发 get 函数

    set 值同样是一个函数，当属性被赋值时，会触发 set 函数

    Vue 是怎么知道数据改变的呢？

    恩，Vue 在 属性的 set 方法中做了手脚，因而当数据改变时，触发 属性的 set 方法，Vue 就能知道数据有改变

    # 依赖收集

    data中的声明的每个属性，都拥有一个数组，保存着谁依赖(使用)了它

    举个例子

    new Vue({    
    data(){        
        return {            
            name:"神仙朱"        
        }    
    }
})

    然后 页面A 引用了name

    <div>{{name}}</div>

    此时，name 把 页面 A 存在它的后宫中（这个页面依赖我）

    为什么呢？

    因为它知道谁依赖它之后，它就可以在发生改变的时候，通知 依赖它的页面，从而让页面完成更新。

    这就是依赖收集，把 依赖了我（使用了我的东西），统统保存起来。

    可是，保存在哪里，具体保存的是什么东西，？？？

    保存的实际上是 页面A的 Watcher。

    总结一下

    1、data 中每个声明的属性，都会有一个 专属的依赖收集器 subs

    2、当页面使用到 某个属性时，页面的 watcher 就会被 放到 依赖收集器 subs 中

    数据 是在什么时候进行 收集依赖 的呢？

    答案是，ObjectdefineProperty- get

    当 页面 A 读取了 name 时，会触发 name 的 get 函数，此时，name 就会保存 页面A 的 watcher 啦！

    # 依赖更新

    每个属性都会有一个专属依赖收集器subs

    # 数据是在什么时候进行依赖更新的呢？

    Object.defineProperty-set

    以上面的例子

    当name改变的时候，name会遍历自己的依赖收集器subs，逐个通知watcher，让watcher完成更新。

    这便可以回答了我开篇的第二个问题，

    Vue 在数据改变时，视图怎么知道什么时候更新?

    在数据变化触发 set 函数时，通知视图，视图开始更新

    # 综上

    1. Object.defineProperty - getter ，用于 依赖收集
    2. Object.defineProperty - setter，用于 依赖更新
    3. 每个 data 声明的属性，都拥有一个的专属依赖收集器 subs
    4. 依赖收集器 subs 保存的依赖是 watcher
    5. watcher 可用于 进行视图更新

    # 总结

    最后，通过解释官方的图来做个总结。

    **1、第一步：**通过 observer 进行数据劫持，然后在需要订阅的地方（如：模版编译），添加观察者（watcher），并立刻通过取值操作触发指定属性的 getter 方法，从而将观察者添加进 Dep （利用了闭包的特性，进行依赖收集），

    **2、第二步：**当data属性发生改变 Setter 触发的时候，就会遍历sub里所有的watcher对象，进行 notify通知给所有观察者并进行相应的 update
