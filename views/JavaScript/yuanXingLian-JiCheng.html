    原型链的继承

    张子涵 8/19/2021 JavaScript

    # 原型

    无论何时，只要创建一个函数，创建一个 prototype 属性（指向原型对象）。默认情况下，所有原型对象自动获得一个名为 constructor 的属性，指回与之关联的构造函数。原型对象包含 constructor 属性和其他后来添加的属性。创建的实例只有一个内部属性指向prototype，但原型上的方法可以调用。

    实例上的属性可以重写原型上的属性（不会修改但会屏蔽），访问一个属性时，会先从实例上搜索属性，若不存在则才会在原型上搜索该属性。

    # 原型链

    通过原型继承多个引用类型的属性和方法。重温一下构造函数、原型和实例的关系：

    • 每个构造函数都有一个原型对象，原型有一个属性指回构造函数，而实例有一个内部指针指向原型。如果原型是另一个类型的实例呢？
    • 那就意味着这个原型本身有一个内部指针指向另一个原型，相应地另一个原型也有一个指针指向另一个构造函数。这样就在实例和原型之间构造了一条原型链。这就是原型链的基本构想。

    ##原型链的继承 让子类的原型对象指向父类的实例，当子类实例找不到对应方法时，会沿着原型链往上找

    ###原型链继承的缺点 多个实例指向同一个原型，一个实例修改其内容其他实例也会改变 无法实现super功能，讲的就是无法对父类传参

    ###构造函数继承

    在子类的构造函数中，执行父类的构造函数，并为其绑定子类的this

    缺点：//虽然解决了多个实例指向同一原型的问题，但并不能够继承父类原型上的方法和属性

    function Parent(name) {
    this.name = [name]
}
Parent.prototype.getName = function () {
    return this.name
}
//在子类的构造函数中，执行父类的构造函数，并为其绑定子类的this
function Child() {
    Parent.call(this,'蔡大大')
}

const child1 = new Child()
const child2 = new Child()
child1.name[0] = '鹿晗'
console.log(child1.name);//鹿晗
console.log(child2.name);//蔡大大
console.log(child2.getName());//not a function
//虽然解决了多个实例指向同一原型的问题
//但并不能够继承父类原型上的方法和属性

    ###组合式继承 既能解决多个实例指向同一原型的问题，又能解决继承父类原型上的方法和属性

    function Parent(name) {
    this.name = [name]
}
Parent.prototype.getName = function () {
    return this.name
}
function Child() {
    Parent.call(this,'蔡大大')
}

Child.prototype = new Parent()
Child.prototype.constructor = Child

const child1 = new Child()
const child2 = new Child()
child1.name[0] = '鹿晗'
console.log(child1.name);//鹿晗
console.log(child2.name);//蔡大大
console.log(child2.getName());//蔡大大

    ###那么问题来了，组合式继承没有缺点吗？当然是有的 每new一个child都声明一次Parent

    ###寄生式组合继承

    function Parent(name) {
    this.name = [name]
}
Parent.prototype.getName = function () {
    return this.name
}
function Child() {
    Parent.call(this,'蔡大大')
}

Child.prototype = Parent.prototype  
Child.prototype.constructor = Child

const child1 = new Child()
const child2 = new Child()
child1.name[0] = '鹿晗'
console.log(child1.name);
console.log(child2.name);
console.log(child2.getName());

    ###缺点 对子类原型的操作（比如增加方法）势必影响父类原型

    ###终极完美boss 浅拷贝~~~

    function Parent(name) {
    this.name = [name]
}
Parent.prototype.getName = function () {
    return this.name
}
function Child() {
    Parent.call(this,'蔡大大')
}

Child.prototype =Object.create(Parent.prototype)   //浅拷贝~~~
Child.prototype.constructor = Child

const child1 = new Child()
const child2 = new Child()
child1.name[0] = '鹿晗'
console.log(child1.name);
console.log(child2.name);
console.log(child2.getName());
